üìã Complete Stringification Pattern Summary

Core Pattern Structure

"[item_name]", [numeric_values]| [more_numeric_values]|| "[rarity_string]" "[body_string]" "[passive_string]"|

Key Components

Component Description Example
Item Name Quoted base item name "classmod_dark_siren"
Numeric Values Level, unknown values, separators 0, 1, 50| 2, 0||
Rarity String Manufacturer.comp_0[rarity]_[rarity_name] "ClassMod.comp_05_legendary"
Body String Specific body/part variant "siren_classmod_leg_01"
Passive String Skill/passive information "passive_blue_1_1_tier_1"

Stringification Rules

  1. All strings must be quoted
  2. Rarity uses comp_0[rarity]_[rarity_name] format
  3. Body parts use specific naming conventions
  4. Passives use passive_[color]_[number]_[number]_tier_[tier] format
  5. Separators are | and , in specific positions

üí° Implementation Examples

For Classmods

  • Use classmod_[character] as base name
  • Rarity: ClassMod.comp_0[rarity]_legendary
  • Body: [character]_classmod_leg_0[number]
  • Passive: passive_[color]_[number]_[number]_tier_[tier]

For Weapons

  • Use [manufacturer]_[type] as base name
  • Rarity: [MANUFACTURER]_[TYPE].comp_0[rarity]_[rarity_name]
  • Specific legendary names for legendary weapons

For Enhancements

  • Use [MANUFACTURER]_Enhancement.comp_0[rarity]_[rarity_name]

For Gadgets

  • Use [manufacturer]_[gadget_type].comp_0[rarity]_[rarity_name]

Complete Working Example

"classmod_dark_siren", 0, 1, 50| 2, 0|| "classmod_dark_siren.ClassMod.comp_05_legendary" "classmod_dark_siren.siren_classmod_leg_01" "classmod_dark_siren.passive_blue_1_1_tier_1"|

This comprehensive system allows for complete stringification of all Borderlands 4 items using the proper game format.

The key insight is using dot notation within quoted strings to combine the base item name with specific components (rarity, body, passive), which avoids the problematic curly bracket part blocks that get filtered out by the decoder.
